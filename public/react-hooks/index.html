react-hooks

发表于 2020-05-18 更新于 2021-08-22 分类于 React

Hook 是 React 16.8 引入的新特性，它可以使我们在不编写类组件的情况下可以去使用 state、生命周期函数、 以及一些其他的 React 功能。并且 Hook 可以从各个组件中提取状态逻辑，以便于代码的复用，并使得样组件之间共享数据就变得很简单。

为什么使用 Hook

React 设计中认为组件是代码复用的最小单元，每个功能都有若干组件组成，并且每个组件都不单纯的孤岛，组件之间是可能会产生联系的，一方面是数据的共享，另一个是功能的复用。

  • 对于组件共享数据，React 采用单向数据流（Flux）进行解决。
  • 对于类组件功能的复用，React 的开发团队也提出过了许多的解决方案，比如早期使用的 Mixins、以及后面提出的 Higher Order Component(简称HOC)、以及 Render Props，可能后面还会有更好的开发理念。

虽然 React 开发团队提出了类组件代码复用的解决方案，但是每个方案都有这自己的问题，在加上难以理解的 class，所以 React 开发团队便推出了 Function Component + Hooks 的开发方式。
但是有一点虽然 React 开发团队提议使用 Function Component + Hooks，但 没有计划从 React 中移除 class。

这样就可以使用 useReducer 来替代 userState 进行组件间的 state 管理。如果需要和子组件共享数据，可以将 useReducer 和 useContext 结合起来，这样所有子组件都可以共享这些数据了。
