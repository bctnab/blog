虚拟DOM

发表于 2021-03-23 更新于 2021-11-09 分类于 JavaScript

如今的前端开发越来越丰富，所用的框架也越来越多，从以前的 React 到后面的 angularJS、Vue等，但无论是那个框架其内部都采取了虚拟DOM、以及 DOM-diff 算法来提升项目运行速度，那么虚拟DOM是个什么东西呢？ DOM-diff 是怎么实现的呢？为什么大多数框架都选择使用它们呢？

四句话

我们可以梳理下DOM-diff的过程：

  1. 使用JS对比描述DOM(虚拟DOM)。
  2. 将虚拟DOM转换为真实DOM并插入界面中(render)。
  3. 如果有事件修改了虚拟DOM，通过 DOM-diff 比较两棵虚拟DOM树的差异，得到差异补丁。
  4. 最后通过补丁，更新真正的DOM。
