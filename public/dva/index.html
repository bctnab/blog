Dva

发表于 2020-08-27 更新于 2021-08-20 分类于 React ， Redux ， Dva

一款整合 redux 和 redux-saga 开发的一款脚手架，其内置了一些 React 项目常用的一些插件和配置，我们只需按照其给出的配置和架构，可以很简单的搭建出我们的项目，我们可以理解为一个轻量级的应用框架。

数据流

内置 redux 与 redux-saga，所以其数据的流向基本是与 redux-saga 相同的，也就是 view 可以通过 dispatch 发送一个 action，用于描述要进行的操作，如果是同步行为会直接通过 Reducers 修改 State，如果是异步行为会先触发 Effects 然后流向 Reducers 最终改变 State。

namespace

为了防止数据的名称覆盖的问题，可以使用 namespace 也就是数据命名空间，如果启用访问数据时需要指定添加命名空间，但是支支持字符串，并不支持通过 . 的方式创建多层命名空间。

reducers

如果使用过 redux 便熟悉此属性，其是唯一一个可以修改 state 的地方。必须通过 dispatch 发送一个 action 才能触发，这里就不过多介绍了。

effects

用于处理异步的 action，但不能修改 state 数据。由于 redux 中 reducers 是不能处理异步请求的，所以如果包含异步逻辑，需要先通过 action 触发 effects，当处理完成异步逻辑后，可以在 effects 中调用 reducers 函数，进行数据的修改。
切记 effects 只能处理异步逻辑，不能用于修改数据。

subscriptions

数据订阅，用于订阅一个数据源，并且数据源不一定是 state 中的数据，还可以是当前的时间、websocket 连接、keyboard 输入、geolocation 变化、history 路由变化等等。

connect

当 Models 定义完成后，我们便可以在组件中使用其定义的数据，只需通过 connect 就可以将自定义组件和 Dva Models 进行连接，但是使用时一定要注意加上 namespace(命名空间)。

总结

Dva 是一款轻量级的应用框架，其内部规定了一系列的规则和配置。其内部还是以前 React + React-Router + Redux + Redux-saga 的架构，但其内部对于各个常用的配置都进行了封装，并规定了项目的架构，这使得项目使用起来更加自动化、也更容易上手和维护。，
